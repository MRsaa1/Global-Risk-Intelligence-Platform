# ℹ️ Информация о Circuit Breakers

## Что это?

Circuit Breakers - это защитный механизм, который **автоматически отключает** неработающие сервисы, чтобы предотвратить каскадные сбои.

## Текущие открытые Circuit Breakers

- ❌ **Neo4j** - Knowledge Graph (граф база данных)
- ❌ **MinIO** - Object Storage (хранилище файлов)
- ❌ **Timescale** - Time-series база данных

## ⚠️ Важно!

**Эти сервисы НЕ обязательны для работы платформы!**

### Что работает БЕЗ них:

✅ **Основная функциональность:**
- Просмотр активов (Assets)
- Аналитика и визуализации
- Command Center
- Dashboard
- Stress Tests
- Все основные страницы

✅ **Основные сервисы:**
- PostgreSQL (основная БД) - ✅ Работает
- Redis (кэш) - ✅ Работает
- API сервер - ✅ Работает
- Web сервер - ✅ Работает

### Что НЕ работает БЕЗ них:

❌ **Knowledge Graph** (Neo4j):
- Анализ зависимостей между активами
- Симуляция каскадных отказов
- Network risk analysis

❌ **File Storage** (MinIO):
- Загрузка BIM файлов
- Хранение отчетов
- Хранение больших файлов

❌ **Time-series** (Timescale):
- Детальная временная аналитика
- Исторические данные с высокой частотой

---

## 🎯 Рекомендация

**Для разработки и тестирования:** Игнорируйте эти предупреждения. Платформа полностью функциональна.

**Для production:** Установите Docker и запустите все сервисы для полной функциональности.

---

## 🔇 Как скрыть предупреждения?

В Command Center есть кнопка "Hide system alerts" - используйте её, если эти предупреждения мешают.
