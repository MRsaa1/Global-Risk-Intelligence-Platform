# Загрузка базы активов (Assets)

В разделе **Assets** платформы хранятся **физические активы** (Physical Assets) и их Digital Twins. Чтобы загрузить много активов сразу, используется **массовый импорт из CSV**.

## Терминология

- **Physical Assets** — физические активы (здания, объекты инфраструктуры, объекты недвижимости).
- **Digital Twin** — цифровой двойник актива; для каждого импортированного актива при необходимости создаётся запись Digital Twin.
- **Массовый импорт (Bulk Import)** — загрузка активов из CSV-файла через API.

## Как загрузить активы

В веб-интерфейсе на странице **Assets** доступны кнопки **Download Template** (скачать шаблон CSV) и **Bulk Import** (загрузить CSV). Ниже — те же шаги через API/curl.

### 1. Скачать шаблон CSV

Шаблон с заголовками и примерами строк:

```bash
curl -o asset_upload_template.csv "http://localhost:8000/api/v1/bulk/assets/template"
```

Или в браузере: `GET /api/v1/bulk/assets/template` (если API поднят).

### 2. Колонки CSV

**Обязательная колонка:**

| Колонка | Описание |
|--------|----------|
| `name` | Название актива (обязательно) |

**Рекомендуемые и опциональные:**

| Колонка | Описание | Пример |
|--------|----------|--------|
| `asset_type` | Тип актива (см. ниже) | office, retail, industrial, data_center |
| `address` | Адрес | Marienplatz 1 |
| `city` | Город | Munich |
| `country_code` | Код страны (2 буквы ISO) | DE, US |
| `latitude` | Широта | 48.1351 |
| `longitude` | Долгота | 11.5820 |
| `valuation` | Оценка стоимости | 125000000 |
| `currency` | Валюта | EUR |
| `year_built` | Год постройки | 2015 |
| `gross_floor_area_m2` | Площадь м² | 25000 |
| `floors_above_ground` | Этажей над землёй | 12 |
| `tags` | Теги через запятую | premium,city-center |
| `description` | Описание | Class A office |

Дополнительно поддерживаются: `region`, `postal_code`. В шаблоне также есть синонимы: `floor_area` вместо `gross_floor_area_m2`, `floors` вместо `floors_above_ground`.

**Типы активов (asset_type)** — значения enum в системе:

- `commercial_office`, `office` — офис
- `commercial_retail`, `retail` — ритейл
- `industrial` — промышленный
- `logistics` — логистика
- `data_center` — ЦОД
- `residential_multi`, `residential` — многоквартирный жилой
- `residential_single` — частный дом
- `infrastructure_transport`, `infrastructure` — транспорт
- `infrastructure_power` — энергоинфраструктура
- `infrastructure_water` — водная инфраструктура
- `energy_solar`, `energy_wind`, `energy_conventional` — энергия
- `healthcare`, `education` — здравоохранение, образование
- `other` — прочее (в т.ч. hotel, mixed_use)

### 3. Проверить файл без импорта (validate)

Перед импортом можно проверить CSV:

```bash
curl -X POST "http://localhost:8000/api/v1/bulk/assets/validate" \
  -F "file=@my_assets.csv"
```

В ответе: количество валидных строк, ошибки по строкам, пример валидных записей.

### 4. Импортировать активы (import)

```bash
curl -X POST "http://localhost:8000/api/v1/bulk/import-assets" \
  -F "file=@my_assets.csv" \
  -F "skip_errors=false" \
  -F "calculate_risks=true"
```

Параметры:

- `file` — CSV-файл (обязательно).
- `skip_errors` — при `true` импортируются только валидные строки, остальные пропускаются; при `false` при наличии ошибок импорт не выполняется.
- `calculate_risks` — после импорта пересчитать риски (climate, physical, network) для созданных активов.

**Ограничения:**

- Размер файла: до 10 MB.
- Количество строк за один запрос: до 1000.

Для больших баз разбейте файл на части по ≤1000 строк и загружайте по одному запросу на файл.

### 5. Альтернативный endpoint загрузки

Тот же функционал доступен по:

```bash
POST /api/v1/bulk/assets/upload
```

Параметры те же: `file`, `skip_errors`, `calculate_risks`.

## Откуда взять данные для активов городов

Подробный гид по источникам (Microsoft Building Footprints, OSM/Geofabrik/Overpass, Google, кадастры, коммерческие базы, синтетические данные) и готовые примеры конвертации в CSV — см. [Полный гид по источникам данных](ASSET_DATA_SOURCES_FULL.md).

Варианты, откуда можно получить или сформировать базу активов с координатами и атрибутами:

1. **OpenStreetMap (OSM)**  
   Экспорт зданий и объектов по региону (Overpass API, QGIS, export.osm). Дальше скриптом преобразовать в наш CSV (name, asset_type, latitude, longitude, при необходимости address, city, country_code, площадь и т.д.).

2. **Государственные реестры и кадастры**  
   Открытые или лицензионные данные по недвижимости/инфраструктуре (в зависимости от страны). Часто есть адрес, тип использования, площадь; геокодирование даст lat/lon.

3. **Коммерческие базы недвижимости и CRE**  
   Коммерческие датасеты по объектам недвижимости (офисы, торговые центры, склады) — обычно уже есть адрес, тип, площадь, иногда оценка. Привести к нашим полям и экспорт в CSV.

4. **Синтетические/демо-данные**  
   Сгенерировать CSV по списку городов: для каждого города — N объектов со случайными координатами в радиусе, типы из списка, площадь и т.д. Удобно для тестов и демо.

5. **Собственные реестры**  
   Экспорт из внутренних систем (ERP, CMMS, реестры активов) в CSV с маппингом колонок на наш шаблон.

Рекомендуемый порядок: скачать шаблон → заполнить или смаппить свои колонки к полям шаблона → проверить через `/assets/validate` → загрузить через `/import-assets` или `/assets/upload`.

## После импорта

- Активы появятся в разделе **Assets** в веб-интерфейсе.
- Для них можно открывать Digital Twin и стресс-тесты.
- При `calculate_risks=true` после импорта пересчитаются риски; иначе их можно пересчитать позже через API или интерфейс.
