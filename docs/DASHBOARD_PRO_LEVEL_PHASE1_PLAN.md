# Dashboard Pro Level — Phase 1: План (Palantir/BlackRock)

## Общие требования (обязательно)

- **Без эмодзи:** нигде в UI — ни в сайдбаре, ни в карточках, ни в статусах, ни в лендингах. Только текст и иконки.
- **Только профессиональные SVG:** иконки — Heroicons (outline/solid) или аналогичные нейтральные SVG; никаких декоративных эмодзи, кастомных «весёлых» иконок или символов вроде флагов/смайлов в интерфейсе.
- **Максимально консервативно и профессионально:** тон интерфейса институциональный: сдержанные цвета, чёткая иерархия, без «игровых» градиентов и анимаций.
- **Бэкап перед стройкой:** перед любыми изменениями по этому плану сделать полный бэкап (например `git checkout -b backup-before-dashboard-pro-level` + коммит текущего состояния, или архив репозитория / `apps/web`). Стройка только после подтверждения бэкапа.

---

## 1. Цветовая схема

- Унифицировать фон везде (один токен, напр. `#09090b`); убрать расхождение с `#0a0e17` в `index.html` / `index.css`.
- Ввести один акцент для действий (напр. синий `#2563eb`) в tailwind и использовать только для активного пункта навигации и основных кнопок.
- Добавить CSS-переменные (`--color-bg`, `--color-accent-primary` и т.д.) в `:root` для возможного ребренда.
- Семантику риска (green/amber/red) не менять; проверить контраст текста (WCAG AA).

**Файлы:** `apps/web/tailwind.config.js`, `apps/web/src/index.css`, `apps/web/index.html`.

---

## 2. Шрифтовая схема

- Зафиксировать единую типографическую шкалу в tailwind (`fontSize`) и иерархию заголовков в `@layer base` (`h1`–`h6`).
- Для чисел и кодов — только `font-mono` + `tabular-nums` (JetBrains Mono); применить в KPI на Dashboard.
- Загрузка шрифтов только в `index.html`; в tailwind/index.css — ссылки на `font-sans` / `font-display` / `font-mono`. Без эмодзи в подписях и лейблах.

**Файлы:** `apps/web/tailwind.config.js`, `apps/web/src/index.css`, при необходимости `apps/web/src/pages/Dashboard.tsx` (KPI).

---

## 3. Быстрая навигация

- Сайдбар: добавить раскладываемый режим с подписями (ширина напр. `w-56`), в свёрнутом — текущий вид (`w-20`, только иконки). Переключатель в сайдбаре.
- В начало Quick Actions на Dashboard добавить пункты Command Center и Dashboard; активный маршрут подсвечивать акцентным цветом (тот же, что в п.1).
- Опционально: тонкая полоса из 5–6 текстовых ссылок над контентом (Command Center, Dashboard, Analytics, Portfolios, Stress, Settings) для быстрого перехода.

**Файлы:** `apps/web/src/components/Layout.tsx`, `apps/web/src/config/quickActions.tsx`, `apps/web/src/components/QuickActionsCards.tsx`.

---

## 4. Аудит и правки: никаких эмодзи, только pro SVG

- Пройти по всем компонентам и страницам, затронутым Phase 1 (Layout, Dashboard, Quick Actions, виджеты Today/Sentiment/Climate Haven, кнопки, статусы).
- Удалить или заменить все эмодзи на нейтральный текст или Heroicons/SVG.
- Убедиться, что все иконки — только SVG (Heroicons или аналогичные), без декоративных символов. Стиль максимально консервативный.

**Файлы:** релевантные компоненты в `apps/web/src` (Layout, страницы, компоненты dashboard, кнопки, бейджи).

---

## Порядок выполнения

0. **Бэкап:** создать ветку/архив, убедиться, что текущее состояние сохранено.
1. Цвета: унификация фона, CSS-переменные, один акцент, применение к активному состоянию навигации.
2. Шрифты: типографическая шкала, иерархия, tabular-nums/mono для KPI.
3. Навигация: расширяемый сайдбар с подписями; Command Center + Dashboard в начало Quick Actions; подсветка активного пункта.
4. Аудит: убрать эмодзи, оставить только профессиональные SVG, консервативный тон.

После выполнения дашборд будет с единой цветовой и шрифтовой схемой, понятной быстрой навигацией, без эмодзи и с максимально консервативным профессиональным видом.
