# Flood Risk Model — где смотреть на фронте и что готово

Краткий гайд: как открыть Municipal Dashboard, вкладка Risk, и что там соответствует пунктам «Flood Risk Model (полный цикл)».

---

## Как открыть

1. **Маршрут:** `/municipal` (страница **Municipal Dashboard**). Модуль CADAPT; при необходимости откроется Access Gate.
2. **Город:** по умолчанию используется `bastrop_tx`. Другой город — через URL: `/municipal?city=houston_tx` или `/municipal?country=US&city=houston_tx`. Данные risk и flood подтягиваются по выбранному городу.
3. **Вкладка:** переключиться на **Risk** (в шапке или через Product modules → Flood).

После перехода на вкладку Risk автоматически запрашиваются:
- `GET /api/v1/cadapt/flood-scenarios?city=...`
- `POST /api/v1/cadapt/flood-risk-product` (body: `{ city, include_grid: true }`)
- `POST /api/v1/cadapt/flood-model/validate-batch`

## Где видна ссылка на USGS

- **В UI:** вкладка **Risk** → блок **Flood Risk Assessment** → справа от заголовка ссылка **«USGS sources (3DEP, WaterWatch)»**. Открывает `/docs/USGS_SOURCES.md` (файл лежит в `apps/web/public/docs/USGS_SOURCES.md`).
- **В репозитории:** [docs/USGS_SOURCES.md](USGS_SOURCES.md) — тот же текст с 3DEP, Circular 1553, EPQS, NWIS, WaterWatch и расширениями (Bulk PQS, TNM Access API).

---

## Что не выведено на фронте

- Карты затопления **по улицам** (построчно).
- **Вероятность по зданиям**.
- Отдельный экран только под гидромодель (HEC-RAS/LISFLOOD-FP не подключены).

---

## Почему не реализовано и можно ли на digital twin

### Гидромодель (HEC-RAS / LISFLOOD-FP)

**Почему не реализовано:**
- **HEC-RAS** — федеральный стандарт USACE, десктоп/CLI; нужна геометрия (створы, русло, сооружения), 1D/2D расчёт, тяжёлый запуск как сервис и лицензии/интеграция.
- **LISFLOOD-FP** — 2D мелководная модель; нужны DEM, осадки, граничные условия; большие объёмы вычислений.
- Сейчас используется **упрощённая** модель уровня города: SCS Curve Number (сток) + Manning (скорость), одна глубина по сценарию на город; сетка для визуализации — простой спад от центра (`flood_hydrology_engine.py`), без полноценной 2D гидравлики.

**На digital twin без полного HEC-RAS/LISFLOOD-FP можно:** уже есть `flood_grid` (массив lat, lon, depth_m) из `POST /flood-risk-product` при `include_grid: true`. Его можно отображать на глобусе/твине (точки или поверхность по глубине). Для «карты по улицам» без 2D-модели — остаётся аппроксимация по этой сетке (интерполяция глубины в произвольной точке).

### Карта затопления и вероятность по зданиям

**Почему не реализовано:**
- **Карта по улицам:** нужна либо 2D гидравлика (LISFLOOD-FP, HEC-RAS 2D и т.п.) с выводом по ячейкам, либо высокодетальный DEM + алгоритм затопления. Сейчас только городская глубина и сетка с радиальным спадом.
- **Вероятность по зданиям:** нужны (1) контуры зданий (footprints) по городу, (2) для каждого здания — пересечение с зоной затопления или глубина в точке/полигоне, (3) вероятность = f(период повторения), например 1/100 в год для 100-летнего события. Слой зданий с координатами по городу в продукте пока не заведён.

**Реализовать на digital twin — да, можно:**
1. **Карта затопления:** использовать `flood_grid` из flood-risk-product и рисовать его на Cesium (в Municipal Dashboard сейчас на глобус уходит только `floodDepthOverride`, полигон берётся из другого API; можно добавить отрисовку именно `flood_grid` — точки/триангуляция по глубине).
2. **Вероятность по зданиям:**  
   - Взять контуры зданий: **OSM** (Overpass API по bbox города) или здания из нашей модели/BIM для объекта.  
   - На бэкенде: для выбранного сценария (10/50/100 yr) по каждому зданию — интерполяция глубины из `flood_grid` в центроиде/полигоне здания; вероятность = 1/return_period (например 0.01 для 100-yr).  
   - На фронте: в Digital Twin (Cesium/BIM) показывать здания с цветом по глубине или по вероятности (например, подсказка при наведении: «Depth 0.8 m, 100-yr probability 1%»).

Итого: полный HEC-RAS/LISFLOOD-FP не обязателен для отображения на твинах. Достаточно текущего flood_grid + слой зданий (OSM или BIM) + интерполяция глубины по зданиям и вывод вероятности из периода повторения — это можно реализовать в рамках текущих digital twin (глобус, панель города, при наличии IFC — и по объекту).

Итог: единый продукт по городу, сценарии 10/50/100, экономика, AEL, источники данных (и ссылка на USGS), валидация и кнопка «View on Globe» — всё подключено и выведено во вкладке **Risk** Municipal Dashboard.
