# Анализ требований «мирового уровня» и вердикт

**Дата:** 2026-02-25  
**Контекст:** Сопоставление вашего списка требований с текущей реализацией Global Risk Platform и рекомендациями по стеку (NVIDIA / Google / Amazon) для модуля «Психологический профиль ЛПР».

---

## 1. Вердикт по основному функционалу

| Требование | Вердикт | Что есть в платформе | Что добавить |
|------------|---------|----------------------|--------------|
| **Единое хранилище (Single Source of Truth)** | ✅ Нужно, частично есть | Layer 0 (Verified Truth), provenance, агрегация геоданных, активы, KG (Neo4j), data federation (Overpass, NIM). Нет «тысяч источников» в реальном времени. | Расширить пайплайны ingestion (биржи, IoT, новости), единый каталог источников и real-time обновления. |
| **Стресс-тестирование и Монте-Карло** | ✅ Нужно, есть | `stress_testing.py`, `universal_stress_engine.py`: Monte Carlo (10k–100k симуляций), Gaussian copula, VaR/CVaR, cascade. Сценарии: credit, market, climate, geopolitical и др. | Доработка сценариев (например «нефть $20 + землетрясение в Тайване» как составной сценарий), больше предустановленных комбо. |
| **Графы и Digital Twin** | ✅ Нужно, есть | Digital Twins (BIM/IFC, состояние, климат, зависимости), Knowledge Graph (Neo4j), Cascade Engine, SCSS/CIP графы зависимостей. | Углубить «скрытые зависимости» (один поставщик/технология) и визуализацию drill-down. |
| **NLP (тональность, мультиязычность)** | ✅ Нужно, почти нет | LLM (NVIDIA) для аналитики и отчётов; нет выделенного модуля тональности новостей/ЦБ/соцсетей. | Модуль NLP: Entity Sentiment, тональность по темам, интеграция новостных лент (GCP Natural Language / AWS Comprehend — см. ниже). |
| **Backtesting** | ✅ Нужно, частично есть | `_get_backtesting_events()` в `stress_report_metrics.py`: региональные кризисы (2008, 2012 Sandy, Harvey, COVID и т.д.), сравнение predicted vs actual. Нет полноценного прогона стратегий по историческим рядам. | Формализовать API backtesting’а «стратегия × исторические кризисы», хранение результатов и метрик. |
| **Early Warning** | ✅ Нужно, есть | SENTINEL (правила, погода, сенсоры, климат), SRO anomaly detection (Z-score, physical-financial coupling), `predictive_ml.generate_early_warning`, триггеры в отчётах. | Настраиваемые триггеры под «аномалии до того, как станут очевидными» (пороги, ML-аномалии). |

**Итог по п.1:** Всё из списка — **нужно**; бóльшая часть уже заложена в платформе. Главные пробелы: масштабный real-time SSOT по тысячам источников, выделенный NLP-слой для новостей/соцсетей, формализованный backtesting стратегий.

## 2. Вердикт по категориям рисков

| Категория | Вердикт | Что есть | Что добавить |
|-----------|---------|----------|--------------|
| **Финансовые и рыночные** | ✅ Нужно | SRO, стресс-тесты (волатильность, ликвидность, процентные ставки, кредитный риск), Universal Stress, PD/LGD. | Явные модули волатильности и ликвидности (данные + метрики). |
| **Геополитика и макро** | ✅ Нужно | Санкции (SCSS: OFAC/EU, screening), геополитические сценарии, исторические события (Ukraine, Brexit, trade wars), country risk. | Граф «5–6 рукопожатий» до подсанкционных лиц; углубление политической нестабильности и разрывов цепочек. |
| **Техногенные и ESG** | ✅ Нужно | Климат (CMIP6, climate risk), кибер (сценарии, триггеры), городская таксономия (compliance, ESG). | Климатические сценарии на 10+ лет, углеродные налоги; кибер — уязвимости и риски утечек. |
| **Операционные и регуляторные** | ✅ Нужно | Compliance (ASGI, BCP, regulatory phrases), audit trail, fraud упомянут в таксономии. | Выделенный фрод-детектор (аномальные транзакции); модуль изменений законодательства по юрисдикциям. |

**Итог по п.2:** Все категории — **нужны**; платформа уже покрывает их частично. Приоритеты: санкционный граф связей, углубление ESG/климат и операционный фрод.

## 3. Вердикт по вашему Must-Have списку

| Пункт | Вердикт | Комментарий |
|-------|---------|-------------|
| **Информационный шум и дезинформация** | ✅ Нужно | Нет в платформе. Логично как отдельный модуль: боты, скоординированные кампании, влияние на панику/акции. Реализация: NLP + соцсети + граф распространения. |
| **Fat Tail / «Чёрный лебедь»** | ✅ Нужно | Стресс-тесты дают тяжёлые сценарии, но явного мониторинга маловероятных катастроф (пандемия, солнечные вспышки, крах интернета) нет. Добавить: каталог tail-событий и триггеры. |
| **Психологический профиль ЛПР** | ✅ Нужно | Не реализован. Ниже — как закрыть решениями NVIDIA / Google / Amazon. |
| **Скрытая концентрация** | ✅ Нужно | Граф зависимостей и cascade есть; «глубокий уровень — один поставщик/технология» можно выводить из KG и SCSS. Нужны отчёты и метрики концентрации по узлам/типам. |
| **Этиомика (первопричины)** | ✅ Нужно | ANALYST и root cause уже есть; цепочки типа «засуха Бразилия → сахар → производители напитков» требуют онтологии причин и связей в KG. Усилить онтологию и вывод цепочек. |

**Итог по п.3:** Все пункты — **нужны**. Платформа ближе всего к «скрытой концентрации» и «первопричинам»; «ЛПР» и «дезинформация» — новые модули.

## 4. Модуль «Психологический профиль ЛПР» — лучшие решения NVIDIA / Google / Amazon

Ваша сводка по стеку совпадает с лучшими практиками. Ниже — как это вписать в текущую платформу и что выбрать.

### 4.1 NVIDIA

| Решение | Задача | Статус в PFRP | Рекомендация |
|---------|--------|----------------|--------------|
| **Riva** | ASR + паралингвистика (интонация, темп, паузы, дрожь) | Используется для TTS/STT (алерты, озвучка отчётов). Паралингвистика для ЛПР не используется. | Добавить пайплайн: аудио выступлений → Riva ASR + извлечение признаков (pitch, pace, pauses) → сохранение в хранилище для LPR. |
| **Maxine** | Микроэкспрессии, стресс, неуверенность по видео | Не используется. | Отдельный сервис/микросервис: видео интервью/выступлений → Maxine → метаданные эмоций/стресса; передавать в единый профиль ЛПР. |
| **NeMo** | Свои LLM под «корпоративный/дипломатический» язык | В проекте: NeMo LLM, Guardrails, Retriever, Agent Toolkit. | Дообучить/адаптировать модель под глоссарий ЛПР и формулировки («рассматриваем все варианты» и т.п.) и использовать в модуле ЛПР. |

### 4.2 Google Cloud

| Решение | Задача | Рекомендация |
|---------|--------|--------------|
| **Vertex AI (Gemini 1.5 Pro)** | Огромное контекстное окно, анализ архива выступлений | Загружать расшифровки + метаданные Riva/Maxine в Gemini: «Как изменилась риторика по Тайваню за N месяцев?». Имеет смысл при гибридной архитектуре (см. ниже). |
| **Natural Language API** | Entity Sentiment по объектам | Использовать для «не просто позитив/негатив, а отношение к конкретным объектам (инновации, регулятор)». Заменить/дополнить любой самописный sentiment. |
| **BigQuery + Trends** | Сопоставление риторики с поиском и рынками | Хранить агрегированную риторику и тренды в BigQuery; дашборды «риторика ЛПР vs поиск/рынок». |

### 4.3 Amazon Web Services

| Решение | Задача | Рекомендация |
|---------|--------|--------------|
| **Comprehend** | Тональность, топики, PII | Автоматическая группировка выступлений по темам и аномалии по темам. Уже есть PII в Ethicist — можно унифицировать. |
| **Rekognition** | Эмоции по видео | Альтернатива/дополнение к Maxine для эмоций по лицу. |
| **S3 + Glue** | Data Lake для видео/аудио/стенограмм | Хранение и каталогизация сырого медиа и производных (стенограммы, метаданные Riva/Maxine, эмбеддинги). |

### 4.4 Таблица выбора по задаче (повтор вашей + привязка к PFRP)

| Задача | Решение | Где в PFRP |
|--------|---------|------------|
| Анализ видео/мимики | NVIDIA Maxine | Новый сервис `lpr_maxine.py` или NIM-контейнер. |
| Анализ архивов выступлений | Google Gemini 1.5 | Сервис анализа риторики (Vertex AI), вызывается из бэкенда или отдельного воркера. |
| Эмоции в голосе | NVIDIA Riva | Расширение `nvidia_riva.py`: не только STT, но и экспорт паралингвистических признаков. |
| Автоматизация потока новостей | AWS Comprehend | Пайплайн новостей → Comprehend (топики + тональность) → БД/озеро; связь с ЛПР по упоминаниям. |
| Предсказание действий | Vertex AI + внутренняя логика | Промпты вида «сравни с доктриной партии, выяви противоречия и смену курса»; выводы писать в профиль ЛПР и в Early Warning. |

### 4.5 Рекомендуемая архитектура (гибрид)

- **Сбор:** AWS (S3, Glue): парсинг новостей, YouTube, Bloomberg; сохранение сырого видео/аудио и метаданных.
- **«Физика»:** NVIDIA Riva (ASR + паралингвистика) + Maxine (микроэкспрессии) → единый слой метаданных по выступлению.
- **Смысл:** Google Vertex (Gemini): текст + метаданные Riva/Maxine, промпты на сравнение с доктриной, выявление противоречий и смены курса.
- **Интеграция в PFRP:**  
  - Профиль ЛПР (новая сущность в БД или KG): идентификатор, выступления, метрики тона/стресса, тренды риторики, флаги «смена курса» / «противоречие».  
  - Early Warning / SENTINEL: подписка на события «смена тона ЛПР по региону X» или «рост стресса в выступлениях по теме Y».  
  - Дашборд и API: например, `/api/v1/lpr/profile/{id}`, `/api/v1/lpr/trends`, экспорт в отчёты.

Такой гибрид хорошо совпадает с вашим описанием и с тем, что уже есть в платформе (NVIDIA, агенты, раннее предупреждение).
