# Как получить «точную копию» 3D модели объекта

На странице актива (Assets → клик по активу) доступны три режима просмотра: **3D View**, **BIM (IFC)**, **BIM (XKT)**. Ниже — как получить реальную 3D модель в каждом режиме.

---

## 1. 3D View (GLB / Digital Twin)

**Источник модели:** Digital Twin Library → привязка модели к активу (GLB).

### Есть ли модели в библиотеке?

- После **Seed** в библиотеке появляются записи каталога. Одна из них — **Demo Duck (GLB)** — уже готова к использованию (публичный GLB), остальные указывают на USD и требуют конвертации (Convert).
- Если библиотека пуста: в модалке **Select Model** нажмите **Seed** → появятся записи, включая демо GLB.

### Как получить 3D в 3D View

1. На странице актива нажмите **Select Model**.
2. Если список пуст — нажмите **Seed** (добавляет каталог, в т.ч. демо GLB).
3. Выберите модель со статусом **Ready (GLB)** (например, **Demo Duck (GLB)**).
4. Нажмите **Attach**.
5. Переключитесь на **3D View** — должна отображаться привязанная модель.

Для записей без GLB: нажмите **Convert** (нужен Celery worker с USD→GLB), после появления GLB — **Attach**.

---

## 2. BIM (IFC)

**Источник модели:** загруженный IFC-файл для данного актива.

### Как получить 3D в BIM (IFC)

1. На странице актива переключитесь на **BIM (IFC)**.
2. Если IFC ещё не загружен — появится экран «No IFC file loaded».
3. Варианты:
   - **Upload from disk** — выберите локальный файл `.ifc` или `.ifczip` (нужен `assetId` у актива).
   - **Demo models** — выберите один из демо-образцов (если доступны по сети).
4. После загрузки модель отображается в BIM (IFC).

Через API: загрузка IFC для актива по эндпоинту `/api/v1/assets/{id}/bim` (см. документацию API).

---

## 3. BIM (XKT)

**Источник модели:** XKT-проект в `public/xeokit-data/projects/` и поле `xkt_project_id` у актива.

### Как получить 3D в BIM (XKT)

1. Положите XKT-данные в каталог:
   - `apps/web/public/xeokit-data/projects/<ProjectId>/`
   - с `index.json` и геометрией (например, `models/.../geometry.xkt`).
2. В данных актива задайте **xkt_project_id** = ID проекта (например, `Duplex` или имя вашей папки).
3. На странице актива выберите **BIM (XKT)** — отобразится модель этого проекта.

Демо: в репозитории уже есть `public/xeokit-data/projects/Duplex/`. Если у актива не задан `xkt_project_id`, показывается демо Duplex с подсказкой.

---

## Итог

| Режим      | Что нужно для реальной модели |
|-----------|--------------------------------|
| **3D View** | Select Model → Seed (если пусто) → выбрать модель Ready (GLB) → Attach |
| **BIM (IFC)** | Загрузить IFC через «Upload from disk» или API для данного актива |
| **BIM (XKT)** | Положить XKT в `public/xeokit-data/projects/<id>/` и задать у актива `xkt_project_id` |
