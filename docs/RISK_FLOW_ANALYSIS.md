# Risk Flow Analysis - Объяснение и интеграция

## 🎯 Что это такое?

**Risk Flow Analysis** (Анализ потока рисков) — это визуализация того, как рисковые события каскадно распространяются через различные секторы экономики и достигают финального уровня воздействия.

### Визуализация: Sankey Diagram (Диаграмма Санкей)

```
[Рисковые события] → [Затронутые секторы] → [Уровень воздействия]
     (Слева)              (В центре)            (Справа)
```

**Пример:**
- **Событие:** Землетрясение (Seismic Event)
- **Секторы:** Инфраструктура → Банки → Недвижимость
- **Воздействие:** Критическое (Critical Impact)

---

## 🔗 Как это связано с остальным проектом?

### 1. **Архитектура: 5 слоев (Five Layers Architecture)**

Risk Flow Analysis является частью **Layer 3: Simulation Engine** (Движок симуляций):

```
Layer 1: Verified Truth (Проверенная правда)
    ↓
Layer 2: Living Digital Twins (Живые цифровые двойники)
    ↓
Layer 3: Simulation Engine ← Risk Flow Analysis здесь!
    ├── Climate Engine (климатические симуляции)
    ├── Economics Engine (экономические модели)
    └── Cascade Engine (каскадные симуляции) ← Основа для Risk Flow
    ↓
Layer 4: Autonomous Agents (Автономные агенты)
    ↓
Layer 5: Protocol (PARS стандарт)
```

### 2. **Cascade Engine (Движок каскадных отказов)**

**Файл:** `apps/api/src/layers/simulation/cascade_engine.py`

**Что делает:**
- Монте-Карло симуляция (10,000 прогонов)
- Моделирует, как отказ одного узла (например, энергосеть) распространяется через сеть зависимостей
- Вычисляет **"скрытый мультипликатор риска"** — насколько традиционные модели недооценивают реальное воздействие

**Пример:**
```
Прямое воздействие: €10M
Каскадное воздействие: €45M
Скрытый мультипликатор: 4.5x
```

### 3. **Knowledge Graph (Граф знаний)**

**Файл:** `apps/api/src/services/knowledge_graph.py`

**Связь:**
- Knowledge Graph хранит зависимости между активами, секторами, регионами
- Cascade Engine использует этот граф для симуляции распространения риска
- Risk Flow визуализирует результаты симуляции

**Пример графа:**
```
Энергосеть (узел A)
    ↓ зависит от
Банки (узел B)
    ↓ зависит от
Недвижимость (узел C)
```

### 4. **Stress Tests (Стресс-тесты)**

**Файл:** `apps/api/src/api/v1/endpoints/stress_tests.py`

**Связь:**
- Stress Tests запускают различные сценарии риска
- Результаты stress test используются для генерации данных Risk Flow
- Risk Flow показывает, как результаты stress test каскадно влияют на портфель

**Пример:**
1. Запускается stress test "Climate Physical Shock"
2. Система вычисляет воздействие на различные секторы
3. Risk Flow визуализирует поток: Climate Event → Real Estate → Banking → Critical Impact

### 5. **Command Center & Dashboard**

**Файлы:**
- `apps/web/src/pages/CommandCenter.tsx` — использует `RiskFlowMini` компонент
- `apps/web/src/pages/Dashboard.tsx` — может показывать Risk Flow в реальном времени
- `apps/web/src/pages/Visualizations.tsx` — полная страница с Risk Flow Analysis

**Интеграция:**
- В Command Center: мини-версия Risk Flow показывается в панелях
- В Dashboard: Risk Flow обновляется в реальном времени при запуске stress tests
- В Visualizations: полная интерактивная визуализация

### 6. **Digital Twin Panel**

**Файл:** `apps/web/src/components/DigitalTwinPanel.tsx`

**Связь:**
- При открытии Digital Twin показывается Risk Cascade Flow
- Показывает, как физические изменения в здании/городе влияют на финансовые риски
- Связывает Layer 2 (Digital Twins) с Layer 3 (Simulation)

## 📊 Компоненты системы

### Frontend компоненты:

1. **`RiskFlowDiagram.tsx`** — основной компонент визуализации
   - Использует Plotly.js для Sankey диаграмм
   - Принимает данные из stress tests
   - Интерактивный (клики на узлы)

2. **`RiskFlowMini.tsx`** — мини-версия для панелей
   - Компактная версия для Command Center
   - Показывается в Digital Twin Panel

3. **`CascadeVisualizer.tsx`** — альтернативная визуализация
   - 3D граф каскадных отказов
   - Использует react-force-graph-3d

### Backend API:

1. **`/api/v1/simulations/cascade`** — запуск каскадной симуляции
   ```python
   POST /api/v1/simulations/cascade
   {
     "trigger_node_id": "power_grid_nyc",
     "failure_threshold": 0.8,
     "time_steps": 12
   }
   ```

2. **`/api/v1/platform/risk-graph`** — получение графа рисков
   - Возвращает связи между активами
   - Используется для построения каскадных путей

## 🔄 Поток данных

```
1. User запускает Stress Test в Command Center
   ↓
2. Stress Test API вызывает Cascade Engine
   ↓
3. Cascade Engine использует Knowledge Graph для поиска зависимостей
   ↓
4. Монте-Карло симуляция (10,000 прогонов)
   ↓
5. Результаты сохраняются в БД
   ↓
6. WebSocket событие отправляется на фронтенд
   ↓
7. Dashboard и Command Center получают событие
   ↓
8. Risk Flow Diagram обновляется с новыми данными
   ↓
9. Пользователь видит визуализацию каскадного воздействия
```

## 💡 Практическое применение

### Пример использования:

**Сценарий:** Банк хочет понять, как землетрясение в Токио повлияет на его портфель

1. **Запуск Stress Test:**
   - Тип: Seismic Event
   - Локация: Tokyo
   - Severity: 0.8

2. **Cascade Engine вычисляет:**
   - Прямое воздействие: €50M (недвижимость в Токио)
   - Каскадное воздействие: €220M (включая зависимости)
   - Скрытый мультипликатор: 4.4x

3. **Risk Flow показывает:**
   ```
   Seismic Event (Tokyo)
       ↓ €50M
   Real Estate Sector
       ↓ €120M
   Banking Sector (кредиты под недвижимость)
       ↓ €50M
   Insurance Sector (страховые выплаты)
       ↓
   Critical Impact: €220M total
   ```

4. **Результат:**
   - Банк видит, что реальный риск в 4.4 раза выше, чем показывают традиционные модели
   - Принимается решение о хеджировании или диверсификации

## 🎨 Визуальные элементы

### Цветовая схема:

- **События (слева):** Теплые приглушенные тона
  - Критические: #8b4c4c (muted red)
  - Высокие: #8b6b4c (muted orange)
  - Средние: #8b834c (muted yellow)

- **Секторы (центр):** Профессиональные синие/бирюзовые
  - Banking: #4a6fa5 (slate blue)
  - Real Estate: #6a8090 (cadet blue)
  - Infrastructure: #5a8a8a (teal)

- **Воздействие (справа):** Градиент от темного к светлому
  - Critical: #7a3d3d (dark muted red)
  - High: #8a5a3d (dark muted orange)

### Ширина потоков:
- Представляет сумму воздействия в миллиардах евро (€B)
- Более толстые потоки = большее финансовое воздействие

## 🔧 Технические детали

### Зависимости:

- **Frontend:**
  - `react-plotly.js` — для Sankey диаграмм
  - `framer-motion` — для анимаций
  - `zustand` — для состояния (через platformStore)

- **Backend:**
  - `numpy` — для математических вычислений
  - `neo4j` — для Knowledge Graph (опционально)
  - `asyncio` — для асинхронных симуляций

### Производительность:

- Cascade Engine: ~100-1000ms для 10,000 прогонов
- Risk Flow визуализация: ~50-200ms для рендеринга
- WebSocket обновления: реальное время (<100ms)
