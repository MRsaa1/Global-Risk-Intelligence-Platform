# Roadmap - Следующие шаги

## 🎯 Приоритетные направления

### 1. **API Gateway - Реализация эндпоинтов** (Высокий приоритет)
- [ ] Реальные эндпоинты для сценариев (CRUD)
- [ ] Эндпоинты для расчетов (запуск, статус, результаты)
- [ ] Эндпоинты для портфелей
- [ ] Интеграция с reg-calculator
- [ ] OpenAPI/Swagger документация

### 2. **Аутентификация и авторизация** (Высокий приоритет)
- [ ] Полная реализация JWT
- [ ] OIDC/SAML интеграция
- [ ] RBAC (роли и права доступа)
- [ ] Управление пользователями
- [ ] Сессии и refresh tokens

### 3. **Мониторинг и Observability** (Высокий приоритет)
- [ ] OpenTelemetry интеграция
- [ ] Prometheus метрики
- [ ] Структурированное логирование
- [ ] Health checks для всех сервисов
- [ ] Tracing для распределенных вычислений

### 4. **Расширение UI** (Средний приоритет)
- [ ] Детальные страницы результатов расчетов
- [ ] Визуализация результатов (графики, таблицы)
- [ ] Экспорт отчетов (PDF, Excel)
- [ ] Фильтры и поиск
- [ ] Настройки пользователя

### 5. **База данных и миграции** (Средний приоритет)
- [ ] SQLAlchemy модели
- [ ] Alembic миграции
- [ ] Seed данные
- [ ] Индексы и оптимизация

### 6. **Расширение функциональности** (Средний приоритет)
- [ ] Планировщик расчетов (scheduler)
- [ ] Уведомления (email, webhooks)
- [ ] История изменений (audit log)
- [ ] Версионирование сценариев

## 🚀 Рекомендуемый порядок реализации

### Этап 1: Критическая инфраструктура (1-2 недели)
1. API Gateway - реальные эндпоинты
2. База данных - модели и миграции
3. Аутентификация - полная реализация

### Этап 2: Observability (1 неделя)
1. Логирование
2. Метрики
3. Tracing

### Этап 3: Расширение функциональности (2-3 недели)
1. Детальные страницы UI
2. Экспорт отчетов
3. Планировщик
