# Инструкция: Синхронизация локальной версии с сервером

## 🎯 Цель

Синхронизировать рабочую локальную версию (http://localhost:5180/command) с production сервером (https://risk.saa-alliance.com).

---

## 🚀 Быстрый способ

### Вариант 1: Использовать скрипт синхронизации (рекомендуется)

```bash
./sync-to-server.sh
```

Этот скрипт:
1. ✅ Создаёт архив текущей версии
2. ✅ Делает backup серверной версии
3. ✅ Копирует новую версию на сервер
4. ✅ Устанавливает зависимости
5. ✅ Перезапускает сервисы

---

## 📋 Что будет синхронизировано

### Включается:
- ✅ Весь исходный код (Python, TypeScript, React)
- ✅ Все конфигурационные файлы
- ✅ Вся документация (включая Strategic Modules)
- ✅ Все новые файлы и изменения

### Исключается:
- ❌ `node_modules/` (переустанавливается на сервере)
- ❌ `.git/` (не нужен на сервере)
- ❌ `__pycache__/`, `*.pyc` (пересоздаётся)
- ❌ `.venv/`, `venv/` (пересоздаётся)
- ❌ `*.db`, `*.sqlite` (база данных на сервере)
- ❌ `.env` (сохраняется существующий на сервере)
- ❌ `backup_*.tar.gz` (локальные backup файлы)

## ⚠️ Важные замечания

### 1. База данных

- Локальная база данных (`dev.db`) **НЕ** копируется на сервер
- На сервере используется отдельная база (`prod.db`)
- Данные на сервере сохраняются

### 2. Environment переменные

- Файл `.env` на сервере **НЕ** перезаписывается
- Если нужно обновить `.env`, сделайте это вручную после синхронизации

### 3. Зависимости

- `node_modules/` и `.venv/` переустанавливаются на сервере
- Это может занять несколько минут

### 4. Downtime

- Во время синхронизации сервисы перезапускаются
- Ожидайте 1-2 минуты downtime

## 🐛 Troubleshooting

### Проблема: Скрипт не может подключиться к серверу

**Решение:**
```bash
# Проверьте SSH подключение
ssh -p 32769 arin@173.212.208.123
```

### Проблема: Сервисы не запускаются

**Решение:**
```bash
# Проверьте логи
ssh -p 32769 arin@173.212.208.123 'tail -50 /tmp/api.log'
ssh -p 32769 arin@173.212.208.123 'tail -50 /tmp/web.log'

# Проверьте, что порты свободны
ssh -p 32769 arin@173.212.208.123 'netstat -tuln | grep -E "9002|5180"'
```

### Проблема: Frontend не собирается

**Решение:**
```bash
# Проверьте Node.js версию на сервере
ssh -p 32769 arin@173.212.208.123 'node --version'

# Попробуйте переустановить зависимости
ssh -p 32769 arin@173.212.208.123 'cd ~/global-risk-platform/apps/web && rm -rf node_modules && npm install'
```

### Проблема: Backend не запускается

**Решение:**
```bash
# Проверьте Python версию
ssh -p 32769 arin@173.212.208.123 'python3 --version'

# Переустановите зависимости
ssh -p 32769 arin@173.212.208.123 'cd ~/global-risk-platform/apps/api && source .venv/bin/activate && pip install -e .'
```
