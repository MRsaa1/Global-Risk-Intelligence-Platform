# 📋 Задачи: База стресс-тестов и исторических событий

## 🎯 Цель

Создать комплексную систему управления стресс-тестами с визуализацией зон риска и детальными отчетами с планами действий.

---

## 📝 Список задач

### 1. База стресс-тестов

#### 1.1 Модели данных для типов стресс-тестов

**Типы стресс-тестов:**
- ✅ Политические (смена режима, санкции, эмбарго)
- ✅ Военные (конфликты, оккупация, блокада)
- ✅ Климатические (наводнения, засухи, ураганы, повышение уровня моря)
- ✅ Финансовые (кризис ликвидности, дефолт, девальвация)
- ✅ Социальные (миграция, демографические изменения)
- ✅ Пандемия (COVID-19, грипп, биологические угрозы)
- ✅ Базель (регуляторные требования, стресс-тесты ЦБ)
- ✅ Протесты (массовые демонстрации, забастовки)
- ✅ Восстания (гражданские конфликты, революции)
- ✅ Массовые беспорядки (гражданские волнения, погромы)

**Поля модели:**
- ID, название, описание
- Тип стресс-теста (enum)
- Географический охват (полигон/зона)
- Параметры интенсивности (severity 0-1)
- Временной горизонт
- Вероятность наступления
- Метрики влияния (PD multiplier, LGD impact, etc.)

### 2. База исторических событий

#### 2.1 Модели данных для исторических событий

**Поля модели:**
- ID, название события
- Дата начала/окончания
- Географический регион
- Тип события (соответствует типам стресс-тестов)
- Описание события
- Фактическое влияние:
  - Пострадавшие активы/предприятия
  - Финансовые потери
  - Время восстановления
  - Каскадные эффекты
- Источники данных
- Уроки и выводы

**Примеры событий:**
- Крым 2014 (военный/политический)
- COVID-19 2020 (пандемия)
- Наводнение в Германии 2021 (климатический)
- Финансовый кризис 2008 (финансовый)
- Протесты в Гонконге 2019 (протесты)
- Арабская весна 2011 (восстания)
- Базель III внедрение (регуляторный)

### 4. Логика определения зон риска

#### 4.1 Алгоритм определения зон

**Для каждого стресс-теста:**
1. Определить географический полигон (координаты границ)
2. Рассчитать радиус влияния на основе:
   - Типа события
   - Интенсивности (severity)
   - Типа активов
   - Исторических данных (если есть похожие события)
3. Разделить на зоны по уровню риска:
   - Зона критического риска (красная)
   - Зона высокого риска (оранжевая)
   - Зона среднего риска (желтая)
   - Зона низкого риска (зеленая)

#### 4.2 Определение активов в зоне

Для каждой зоны:
- Найти все активы в радиусе (PostGIS query)
- Классифицировать по типу:
  - Предприятия (коммерческие, промышленные)
  - Здания (офисные, жилые, инфраструктура)
  - Учреждения (банки, больницы, школы)
  - Инфраструктура (мосты, дороги, энергосети)
  - Военные объекты (базы, склады, полигоны)
  - Строящиеся объекты (девелоперские проекты)

#### 4.3 Определение влияния на организации

Для каждого актива определить:
- Владельца/оператора (девелопер, банк, страховщик, военные, предприятие)
- Тип влияния (финансовое, операционное, репутационное)
- Степень влияния (критическое, высокое, среднее, низкое)
- Время до начала влияния (немедленно, 24ч, 72ч, неделя+)

### 6. Детальная карта зоны

#### 6.1 Компонент детальной карты

**При клике на зону:**
- Открывается модальное окно/панель с детальной картой
- Показываются все активы в зоне:
  - Маркеры на карте
  - Список активов с фильтрацией
  - Группировка по типам:
    - 🏢 Предприятия
    - 🏛️ Учреждения
    - 🏦 Банки
    - 🏥 Больницы
    - 🏫 Школы
    - 🏭 Промышленные объекты
    - 🌉 Инфраструктура

#### 6.2 Детальная информация об активе

**Для каждого актива:**
- Название, тип, адрес
- Владелец/оператор (девелопер, банк, страховщик, военные, предприятие)
- Текущий риск-скор
- Прогнозируемое влияние:
  - Вероятность повреждения
  - Ожидаемые потери
  - Время простоя
  - Каскадные эффекты
- Влияние на владельца:
  - Финансовое (для девелоперов, страховых, банков)
  - Операционное (для предприятий, военных)
  - Репутационное
- Кнопка "Подробно" → полный отчет с планом действий для владельца

### 7. Генерация отчетов

#### 7.1 Структура отчета

**Разделы отчета:**

1. **Резюме**
   - Тип стресс-теста
   - Географический охват
   - Общая оценка риска
   - Ключевые метрики

2. **Зоны риска**
   - Карта с зонами
   - Описание каждой зоны
   - Количество активов по зонам

3. **Пострадавшие активы**
   - Список активов с детализацией
   - Группировка по типам
   - Финансовое влияние

4. **Анализ влияния**
   - Прямое влияние
   - Каскадные эффекты
   - Время восстановления
   - Исторические аналогии

5. **Влияние на организации** ⭐
   
   **5.1 Девелоперы (разработчики недвижимости)**
   - Финансовое влияние:
     - Снижение стоимости активов в зоне риска
     - Задержки строительства/реконструкции
     - Потеря арендаторов/покупателей
     - Увеличение затрат на страхование
     - Проблемы с финансированием проектов
   - Операционное влияние:
     - Остановка строительных работ
     - Эвакуация персонала и оборудования
     - Повреждение строящихся объектов
     - Потеря доступа к объектам
   - Репутационные риски:
     - Снижение доверия инвесторов
     - Проблемы с продажей объектов
   - План действий для девелоперов:
     - Оценка уязвимости проектов в зоне риска
     - Диверсификация портфеля (географическая)
     - Усиление страхового покрытия
     - Создание резервных фондов на случай событий
     - Планы эвакуации и защиты строящихся объектов
     - Пересмотр сроков и бюджетов проектов
     - Коммуникация с инвесторами и стейкхолдерами
   
   **5.2 Страховые компании**
   - Финансовое влияние:
     - Массовые страховые выплаты (claims)
     - Резкое увеличение убытков
     - Проблемы с перестрахованием
     - Необходимость увеличения резервов
     - Падение прибыльности портфеля
   - Операционное влияние:
     - Рост количества заявок на выплаты
     - Необходимость быстрой оценки ущерба
     - Проблемы с ликвидностью для выплат
     - Увеличение нагрузки на персонал
   - Регуляторное влияние:
     - Требования к увеличению капитала
     - Пересмотр моделей риска
     - Усиление надзора регуляторов
   - План действий для страховых:
     - Пересмотр страховых тарифов в зонах риска
     - Ограничение покрытия в высокорисковых зонах
     - Увеличение резервов на убытки (IBNR)
     - Усиление перестрахования
     - Стресс-тестирование портфеля
     - Быстрая мобилизация команды по урегулированию убытков
     - Коммуникация с перестраховщиками
     - Обновление моделей ценообразования
   
   **5.3 Военные структуры**
   - Операционное влияние:
     - Необходимость защиты критической инфраструктуры
     - Эвакуация военных объектов в зоне риска
     - Обеспечение безопасности населения
     - Координация с гражданскими службами
   - Финансовое влияние:
     - Увеличение расходов на операции
     - Повреждение военной инфраструктуры
     - Потеря оборудования и техники
   - Стратегическое влияние:
     - Изменение дислокации сил
     - Пересмотр планов обороны
     - Координация с союзниками
   - План действий для военных:
     - Оценка угрозы для военных объектов
     - Планы защиты критической инфраструктуры
     - Протоколы эвакуации и передислокации
     - Координация с гражданскими властями
     - Планы обеспечения безопасности населения
     - Резервные планы связи и логистики
     - Оценка готовности к реагированию
   
   **5.4 Банки**
   - Финансовое влияние:
     - Увеличение дефолтов по кредитам
     - Снижение стоимости залогов (недвижимость)
     - Проблемы с ликвидностью
     - Необходимость увеличения резервов
   - План действий для банков:
     - Пересмотр кредитных портфелей в зоне риска
     - Увеличение резервов на возможные потери
     - Стресс-тестирование портфеля
     - Реструктуризация проблемных кредитов
   
   **5.5 Предприятия**
   - Операционное влияние:
     - Остановка производства
     - Потеря доступа к объектам
     - Проблемы с цепочками поставок
     - Эвакуация персонала
   - План действий для предприятий:
     - Планы эвакуации и защиты
     - Резервные производственные мощности
     - Диверсификация поставщиков
     - Страхование бизнес-прерывания

6. **План действий** ⭐
   - Для каждой структуры/организации (детали выше)
   - Приоритизация действий по срочности
   - Временные рамки (немедленно, 24ч, 72ч, неделя, месяц)
   - Ответственные лица/отделы
   - Зависимости между действиями
   - Метрики успеха выполнения плана

7. **Рекомендации**
   - Краткосрочные (0-3 месяца)
   - Среднесрочные (3-12 месяцев)
   - Долгосрочные (1+ год)
   - Специфичные рекомендации для каждого типа организации

### 8. Примеры планов действий по типам организаций

#### 8.1 Девелоперы - Наводнение в долине Рейна

**Немедленные действия (0-24 часа):**
1. Остановить все строительные работы в зоне риска
2. Эвакуировать персонал и оборудование
3. Защитить строящиеся объекты (укрепление, барьеры)
4. Уведомить инвесторов о ситуации
5. Оценить ущерб на объектах в стадии строительства

**Краткосрочные (1-7 дней):**
1. Провести детальную оценку ущерба
2. Пересмотреть сроки сдачи проектов
3. Усилить страховое покрытие для оставшихся объектов
4. Создать резервный фонд на случай задержек
5. Коммуникация с арендаторами/покупателями

**Среднесрочные (1-3 месяца):**
1. Восстановление поврежденных объектов
2. Пересмотр портфеля проектов (диверсификация)
3. Пересмотр бюджетов с учетом рисков
4. Разработка планов защиты от будущих событий

#### 8.2 Страховые компании - Массовые выплаты

**Немедленные действия (0-24 часа):**
1. Мобилизовать команду по урегулированию убытков
2. Оценить общий объем потенциальных выплат
3. Проверить ликвидность для выплат
4. Уведомить перестраховщиков
5. Открыть горячую линию для клиентов

**Краткосрочные (1-7 дней):**
1. Ускорить обработку заявок на выплаты
2. Увеличить резервы на убытки (IBNR)
3. Ограничить новое покрытие в зоне риска
4. Пересмотреть тарифы для зоны риска
5. Координация с перестраховщиками

**Среднесрочные (1-3 месяца):**
1. Обновить модели ценообразования
2. Пересмотреть портфель покрытий
3. Усилить перестрахование
4. Стресс-тестирование капитала
5. Коммуникация с регуляторами

#### 8.3 Военные структуры - Защита критической инфраструктуры

**Немедленные действия (0-24 часа):**
1. Оценить угрозу для военных объектов
2. Усилить охрану критической инфраструктуры
3. Подготовить планы эвакуации военных объектов
4. Координация с гражданскими властями
5. Мобилизация резервных сил при необходимости

**Краткосрочные (1-7 дней):**
1. Передислокация уязвимых объектов
2. Обеспечение безопасности населения
3. Защита военных складов и баз
4. Резервные планы связи и логистики
5. Координация с союзниками

**Среднесрочные (1-3 месяца):**
1. Пересмотр планов обороны региона
2. Укрепление военной инфраструктуры
3. Обновление протоколов реагирования
4. Учения по реагированию на подобные события

## 🧠 NVIDIA PyG для Graph Neural Networks

**Контейнер:** `nvcr.io/nvidia/pyg:25.11-py3`

**Что включено:**
- PyTorch Geometric — графовые нейронные сети
- PyTorch — обучение моделей
- cuGraph — GPU-ускоренные графовые алгоритмы
- RAPIDS — быстрая обработка данных
- CUDA — GPU ускорение

**Применение в проекте:**

| Use Case | Описание |
|----------|----------|
| **Cascade Prediction** | GNN предсказывает распространение риска по графу активов |
| **Risk Propagation** | Моделирование, какие активы пострадают и в каком порядке |
| **Early Warning** | Предсказание, куда распространится риск при событии |
| **Network Analysis** | Определение критических узлов и зависимостей |
| **Impact Estimation** | Оценка влияния события на связанные активы |

**Интеграция:**
```
Neo4j (Knowledge Graph)
    ↓ экспорт графа
PyG (Graph Neural Network)
    ↓ обучение на исторических каскадах
GNN Model
    ↓ предсказание
Cascade Predictions → SENTINEL Agent → Alerts
```

**Модели GNN для проекта:**
- **GraphSAGE** — для больших графов с миллионами узлов
- **GAT (Graph Attention)** — для weighted edges (разная важность связей)
- **GCN (Graph Convolution)** — для классификации узлов по риску

**Docker интеграция:**
```yaml
# docker-compose.nvidia.yml
pyg:
  image: nvcr.io/nvidia/pyg:25.11-py3
  container_name: pfrp-pyg
  deploy:
    resources:
      reservations:
        devices:
          - driver: nvidia
            count: 1
            capabilities: [gpu]
  volumes:
    - ./models:/workspace/models
    - ./data:/workspace/data
  ports:
    - "8888:8888"  # JupyterLab
```
