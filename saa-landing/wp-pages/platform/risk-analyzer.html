About

SAA Risk Analyzer is a full-featured portfolio risk analytics service: a single Go backend and React web interface cover risk metrics, portfolio management, market data, and secure access. The risk engine (Gonum) computes VaR/CVaR using historical and parametric methods, correlations, PCA, stress-test scenarios, and backtesting; market data is pulled from multiple APIs with automatic fallback; authentication is JWT, passwords are Argon2id, roles are RBAC.

Capabilities

Risk analytics

  • VaR — Value at Risk: historical and parametric methods, 95% / 99% / 99.9%, 1- and 10-day horizon
  • CVaR — Conditional VaR (Expected Shortfall), tail risk estimate
  • Correlations — Asset correlation matrix, interactive heatmap
  • PCA — Principal Component Analysis for factor risk decomposition
  • Stress tests — Scenarios: crisis, market crash, recession; configurable shocks
  • Backtesting — VaR model validation (Kupiec POF, Christoffersen)
  • Risk dashboard — VaR, CVaR, volatility, asset contribution to risk

Portfolios and data

  • Portfolios — Create, edit, delete; multiple portfolios
  • Positions — Add, update, remove positions by asset
  • Market data — CoinGecko, Binance, Yahoo Finance with automatic fallback when a source is unavailable
  • Import — Upload positions and prices (CSV), bulk update

Security and infrastructure

  • Authentication — JWT (access + refresh), stored in localStorage
  • Passwords — Argon2id (time=1, memory=64MB), 16-byte salt
  • RBAC — Roles: admin, analyst, viewer
  • CORS, rate limiting — Configurable policies
  • Logging — Structured logs (Zap), levels and format in config

Technology stack

Backend

Go 1.22, Gin, GORM, Gonum, JWT (golang-jwt), Argon2id (crypto), Viper, Zap

Frontend

React 18, TypeScript 5.2, Vite 5, Tailwind CSS, Zustand, Recharts, Framer Motion, Axios

Database

PostgreSQL 15

Proxy (optional)

Caddy 2 or Nginx

Architecture

Browser (React SPA, Vite dev: 5173 / Docker: 3001) talks to the Backend (Go + Gin on :8084) over HTTP/REST. Backend exposes handlers for portfolio, risk, health; services for risk (VaR, CVaR, correlation, PCA, stress, backtest, returns) and market data (multi-API fallback); auth (JWT, Argon2id, RBAC) and jobs. PostgreSQL (risk_db) stores assets, prices, portfolios, positions, jobs via GORM.

Quick start

Requirements: Go 1.22+, Node.js 18+, PostgreSQL 15+. Clone repo, run go build in backend and npm install in frontend; create database risk_db; set env (DB_*, PORT, JWT_SECRET); start backend and frontend. Docker: docker-compose up -d --build. API: http://localhost:8084, Frontend: http://localhost:3001.

API (main endpoints)

Base URL: /api

  • GET /health — Service health
  • GET/POST/PUT/DELETE /portfolios, /portfolios/:id — Portfolio CRUD
  • POST/PUT/DELETE /portfolios/:id/positions — Positions
  • GET /market/price/:symbol — Current price
  • GET /risk/dashboard?portfolio_id= — Risk dashboard
  • POST /risk/var, /risk/cvar, /risk/correlation — VaR, CVaR, correlation
