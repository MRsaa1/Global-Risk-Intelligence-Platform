About

SAA Risk Analyzer is a full-featured portfolio risk analytics service: Go backend (Gin, GORM, Gonum) and React frontend cover risk metrics, portfolio management, market data, and secure access. The risk engine computes VaR/CVaR (historical, parametric, and Monte Carlo with 10k–100k paths), correlations, PCA, stress-test scenarios, backtesting, and portfolio optimization. GPU Service (Python, cuDF, CVXPY) handles Monte Carlo, covariance, and optimization; AI Service (NVIDIA NIM, RAG) provides explain and document intelligence. Results can be exported to ARIN for a unified risk verdict. Market data from multiple APIs with fallback; JWT, Argon2id, RBAC. Production: analyzer.saa-alliance.com.

Capabilities

Risk analytics

  • VaR / CVaR — Historical, parametric, and Monte Carlo (10k–100k paths); 95% / 99% / 99.9%, 1- and 10-day horizon
  • Portfolio optimization — Efficient frontier, covariance (GPU-accelerated via GPU Service)
  • Correlations — Asset correlation matrix, interactive heatmap
  • PCA — Principal Component Analysis for factor risk decomposition
  • Stress tests — Scenarios: crisis, market crash, recession; configurable shocks
  • Backtesting — VaR model validation (Kupiec POF, Christoffersen)
  • Risk dashboard — VaR, CVaR, volatility, asset contribution to risk
  • Explain — AI Service (NVIDIA NIM, RAG) for narrative and document intelligence
  • Export to ARIN — Send portfolio/risk context for unified verdict from the 13-agent council

Portfolios and data

  • Portfolios — Create, edit, delete; multiple portfolios
  • Positions — Add, update, remove positions by asset
  • Market data — CoinGecko, Binance, Yahoo Finance with automatic fallback when a source is unavailable
  • Import — Upload positions and prices (CSV), bulk update

Security and infrastructure

  • Authentication — JWT (access + refresh), stored in localStorage
  • Passwords — Argon2id (time=1, memory=64MB), 16-byte salt
  • RBAC — Roles: admin, analyst, viewer
  • CORS, rate limiting — Configurable policies
  • Logging — Structured logs (Zap), levels and format in config

Technology stack

Backend

Go 1.22+, Gin, GORM, Gonum; JWT (golang-jwt), Argon2id (crypto), Viper, Zap. Handlers for portfolio, risk, health; services for VaR, CVaR, correlation, PCA, stress, backtest, returns, market data.

Frontend

React 18, TypeScript 5.2, Vite 5, Tailwind CSS, Zustand, Recharts, Framer Motion, Axios

GPU Service

Python: cuDF, CVXPY. Monte Carlo (10k–100k paths), covariance, portfolio optimization. GPU-accelerated when hardware available.

AI Service

NVIDIA NIM, RAG. Explain (narrative, document intelligence). Integrates with Risk Analyzer for interpretable risk outputs.

Database

PostgreSQL 15

Proxy (optional)

Caddy 2 or Nginx

Architecture

Browser (React SPA) talks to the Go backend (Gin) over HTTP/REST. Backend orchestrates: Data (market, portfolios), Risk Engine (VaR, CVaR, PCA, stress, backtest), GPU Service (Monte Carlo, optimization, covariance), AI Service (NVIDIA NIM, RAG for explain), Export (to ARIN). PostgreSQL stores assets, prices, portfolios, positions. Auth: JWT, Argon2id, RBAC.

Quick start

Requirements: Go 1.22+, Node.js 18+, PostgreSQL 15+. Clone repo, run go build in backend and npm install in frontend; create database risk_db; set env (DB_*, PORT, JWT_SECRET); start backend and frontend. Docker: docker-compose up -d --build. API: http://localhost:8084, Frontend: http://localhost:3001.

API (main endpoints)

Base URL: /api

  • GET /health — Service health
  • GET/POST/PUT/DELETE /portfolios, /portfolios/:id — Portfolio CRUD
  • POST/PUT/DELETE /portfolios/:id/positions — Positions
  • GET /market/price/:symbol — Current price
  • GET /risk/dashboard?portfolio_id= — Risk dashboard
  • POST /risk/var, /risk/cvar, /risk/correlation — VaR, CVaR, correlation
