About

ARIN Platform is the multi-agent AI council at the core of SAA Alliance. 13 specialized agents (Credit, Market, Operational, Liquidity, Regulatory, Systemic, Narrative, Counterparty, Model Risk, ESG, Physical Asset, Sentinel, Ethicist) independently analyze each input and produce a single explainable verdict (BUY/SELL/HOLD/AVOID) with consensus, dissent, and confidence. Meta-Decision Governor, Entity Memory, Learning Agent; Product Accelerators. Graph analysis (dependencies, cascade, GNN for systemic risk), LLM reasoning via NVIDIA NIM (DeepSeek R1, Llama 3.3 70B, Llama 3.1 8B/70B, Nemotron), NeMo Guardrails; GPT-4 fallback. Auth, compliance, full audit trail for regulators. arin.saa-alliance.com

Capabilities

Risk agents and AI

  • 13 specialized agents — Credit, Market, Operational, Liquidity, Regulatory, Systemic, Narrative, Counterparty, Model Risk, ESG, Physical Asset, Sentinel, Ethicist; each with its own perspective; NVIDIA NIM (DeepSeek R1, Llama 3.3 70B, etc.), NeMo Guardrails
  • Meta-Decision Governor, Entity Memory, Learning Agent — Consensus aggregation, dissent tracking, confidence score; persistent entity context; continuous improvement
  • Graph analysis — Dependencies, cascade effects, GNN for systemic risk (PyTorch Geometric, DGL)
  • LLM and ML — Response caching, retry, multi-provider; XGBoost for credit; GNN for network analysis

Unified integration and export

  • Unified Integration — Single data feed from all six SAA Alliance modules (Risk Analyzer, Investment Dashboard, News Analytics, Crypto Analytics, Global Risk Platform); final verdict (BUY/SELL/HOLD/AVOID) and risk score with full audit trail
  • Export API — Accept exports from Risk Analyzer, Investment, News, Crypto, Global Risk; persistent storage, entity binding (portfolio, symbol)

Auth, compliance, and infrastructure

  • Authentication — Registration, login, JWT, OAuth; users and roles in PostgreSQL (Alembic migrations)
  • Compliance — GDPR (data access, export, deletion), audit log, retention policies, backup and recovery
  • Visualizations — Graphs (2D/3D force-graph), ECharts, D3.js, Plotly 3D, Three.js, Recharts; risk dashboards
  • Integrations — NVIDIA Build panel, Superset, portfolio sync, alerts, config storage (NVIDIA, Superset)
  • Infrastructure — Cache (Redis), queues (Celery), monitoring, health checks, Docker/Kubernetes

Architecture

Top layer: Auth (JWT, OAuth), Compliance (GDPR, Audit, Backup), Config API. Orchestrator distributes tasks across 13 agents. Core risk: Credit, Market, Operational, Liquidity, Regulatory, Systemic; additional: Narrative, Counterparty, Model Risk, ESG, Physical Asset, Sentinel, Ethicist. Meta-Decision Governor aggregates views into Decision Object; Entity Memory and Learning Agent support context and adaptation. Graph Builder (dependencies, clusters, paths) + GNN (PyTorch Geometric, DGL). AI Engine: LLM Manager (NVIDIA NIM, OpenAI fallback), NeMo Guardrails, ML models. Unified: Aggregator, Verdict Generator (BUY/SELL/HOLD/AVOID), Export Storage, API keys. Data: PostgreSQL (users, roles), Redis (cache), Neo4j (graph).

Technology stack

Backend

Python 3.11+, FastAPI, Uvicorn, Pydantic 2; SQLAlchemy 2.x, asyncpg, Alembic; PostgreSQL. Redis, Celery. Neo4j, NetworkX, PyVis. JWT (python-jose), Passlib (bcrypt), OAuth-ready.

AI / ML

LLM: NVIDIA NIM (DeepSeek R1, Llama 3.3 70B, Llama 3.1 8B/70B, Nemotron), NeMo Guardrails; GPT-4 fallback. ML: XGBoost, PyTorch, PyTorch Geometric / DGL (GNN). Data: pandas, numpy, polars, scipy.

Frontend

Next.js 14, React 18, TypeScript. Tailwind CSS. ECharts, D3.js, Plotly, Three.js, Recharts, react-force-graph-2d/3d. Zustand, TanStack React Query, Axios.

Infrastructure

Docker, Docker Compose, Kubernetes. CORS, optional nginx, SSL.

Quick start

Requirements: Python 3.11+ (3.12 recommended), PostgreSQL 15+, Node.js 18+. Clone arin-platform, run ./DO_EVERYTHING.sh (DB, .env, venv, migrations). Backend: uvicorn backend.main:app --reload → API http://localhost:8000, docs http://localhost:8000/docs. Frontend: cd frontend && npm install && npm run dev → http://localhost:3000. Docker: docker-compose up -d.

Main API

Base: /api/v1

  • Health — GET /health, GET /api/v1/performance/health
  • Agents — GET /api/v1/agents, GET /api/v1/agents/{id}
  • Analysis — POST /api/v1/risks/analyze (market_analysis, credit_analysis, liquidity_analysis, …)
  • Graph — GET /api/v1/graph, GET /api/v1/graph/visualization
  • Auth — POST /api/v1/auth/register, POST /api/v1/auth/login, GET /api/v1/auth/me
  • Unified — POST /api/v1/unified/export, GET /api/v1/unified/exports/{entity_id}, GET /api/v1/unified/verdict/{entity_id}
  • Alerts — GET /api/v1/alerts
  • LLM — POST /api/v1/llm/generate, POST /api/v1/llm/extract
