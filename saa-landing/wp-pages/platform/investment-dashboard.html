About

Professional platform for analysis of major US equities with BlackRock-style HTML reports. The service grew from a simple report generator into a full ecosystem: API, real-time analytics, ML valuation, risk, ESG, streaming, and monitoring.

Capabilities

Reports and analytics

  • Institutional reports V2 — BlackRock-style HTML with Executive Summary, scores, and visualizations
  • SWOT, technical and fundamental analysis — RSI, MACD, volume, key metrics
  • Valuation — DCF, EV/Sales, peer multiples, SOTP, Vanguard-style methodology
  • Risk analysis — Risk matrix (probability/impact), Risk Radar, scoring
  • Enhanced analyst consensus — Rating distribution, target prices, momentum
  • Extended data — TradingView-style, segments, geography, seasonality
  • Bilingual reports — EN/RU, multilingual translations

Data and infrastructure

  • Multiple sources — yfinance, SEC EDGAR (10-K/10-Q), segment loading, CDN
  • Cache and storage — Redis, PostgreSQL, DuckDB for analytics
  • Validation and quality — Data validation, ESG scoring, compliance

ML and AI (optional)

  • NVIDIA modules — RAG for peer comparison, DCF reasoning, report generation, translations
  • Signals and portfolio — Signal extraction from news/reports, portfolio optimization (Markowitz)
  • Forecasts and sentiment — Price predictor, sentiment analyzer

Platform and DevOps

  • Web API — Flask (stable_server) + FastAPI (app/api/v1), health, metrics, RBAC
  • Streaming — Kafka producer/consumer, stream processor
  • Monitoring — Prometheus, Grafana, structlog, health checks
  • Deploy — Docker Compose (web, Redis, Postgres, Nginx, Prometheus, Grafana, Kafka, Airflow, ELK), Kubernetes (deployment, services, ingress, HPA, RBAC)

Technology stack

Backend

Python 3.9+, Flask 3, FastAPI, Uvicorn, Gunicorn. pandas, numpy, yfinance, scikit-learn, scipy.

Database and cache

PostgreSQL (SQLAlchemy, Alembic), DuckDB, Redis (hiredis). Prometheus, Grafana, structlog.

Security and infra

python-jose, passlib/bcrypt, RBAC, compliance. Docker, Docker Compose, Kubernetes. Kafka, Zookeeper. Apache Airflow (in docker-compose). Elasticsearch, Kibana (optional).

Production

Gunicorn WSGI, port 5001. Report version: V2 (Institutional). Health: GET /health, GET /api/report_version.

API (main)

Base: /api

  • GET / — Web UI, ticker selection and report generation
  • GET /health — Server status
  • GET /api/report/<ticker> — HTML report. Params: view, v2, download, lang (en/ru)
  • GET /api/market_overview — Market overview, top valuations
  • POST /api/analyze — Bulk analysis: {"tickers": ["AAPL", "MSFT"]}
  • GET /api/report_version — Report version (v2, Institutional)

Report structure V2

Executive Summary; Quantitative Scores (Valuation, Growth, Quality, Momentum, Risk, Overall); Investment Summary; Valuation Data (multiples, DCF, EV/Sales, SOTP); Technical Analysis (chart, RSI, MACD, volume); Enhanced Analyst Consensus; Advanced Visualizations (Valuation Heatmap, Risk Radar, Peer Benchmarking); Risk Analysis; Peer Comparison; Extended Data & Analytics; Latest News; Financial Statements; SWOT; Professional Trade Idea.

Quick start

Requirements: Python 3.9+, 4GB+ RAM. pip install -r requirements.txt. Optional .env: TELEGRAM_*, ALPHA_VANTAGE_API_KEY, REDIS_URL, POSTGRES_URL. Run python stable_server.py → http://localhost:5001. CLI: python main.py.
