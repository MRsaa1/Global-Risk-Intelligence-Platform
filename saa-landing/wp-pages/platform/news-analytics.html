About

News Analytics Portal is the financial news intelligence module of SAA Alliance. 12 specialized risk agents; integration with the ARIN Council for unified verdict. Features: RiskMirror, Trust Heatmap, Headline-to-PnL; Magic Screen, Data Flywheel, BigQuery; Impact Graph (Neo4j or SQLite). AI: NVIDIA NIM (Llama 3.3 70B), NeMo Guardrails, Nemotron NER. Stack: FastAPI, SQLite, Redis. Dashboard, impact graph, multilingual support, export to Telegram and Excel. Production: Python API on the server (news.saa-alliance.com); optionally a Go dashboard for local or alternative builds.

Production stack (server)

Live: news.saa-alliance.com. Internal port 18080, nginx HTTPS/WebSocket.

Component Technology
OS Ubuntu (Linux 6.8), x86_64
Web nginx (HTTPS, Let's Encrypt), HTTP/2, WebSocket
API Python 3.12, FastAPI, uvicorn (12 workers)
Port 18080 (internal), 443 (news.saa-alliance.com)
DB SQLite (signals.db); optionally Neo4j (Bolt 7687, HTTP 7474) for graph
Cache Redis (LLM response caching: REDIS_ENABLED, REDIS_HOST/PORT/DB)
LLM / AI NVIDIA API only: integrate.api.nvidia.com, default model meta/llama-3.3-70b-instruct, key NVIDIA_API_KEY
Impact graph With Neo4j — ImpactGraphServiceNeo4j (2–3 level traversal); else ImpactGraphServiceV2 (SQLite, 1 level); NetworkX for metrics
Extras Nemotron NER, optional nvidia_retriever, nvidia_ocr, NeMo Guardrails
Integrations Telegram (digests, alerts), Excel (openpyxl), PDF (reportlab), email digest

Deployment on server

  • Service: signal-analysis-api.service (systemd)
  • Path: /var/www/signal-analysis-api
  • Command: venv/bin/python3 -m uvicorn app:app --host 0.0.0.0 --port 18080 --workers 12
  • Config: .env (including NVIDIA_API_KEY). Logs: nohup.out, app.log

Capabilities (Python app)

  • Ingest: RSS feeds, HTML parsing; deduplication, date and source-trust normalization
  • Risk analysis: Each item through NVIDIA API (Llama 3.3 70B): summary, label, impact, confidence; responses cached in Redis
  • Analysis generation: POST /generate-analysis/{signal_id} — NVIDIA API; result stored in SQLite
  • Impact graph: Entities and relations per article; with Neo4j — 2–3 level traversal; else SQLite; NetworkX for metrics
  • API: Dashboard, health, stats, signals (with filters), single signal, generate-analysis, telegram-digest, export (HTML/Excel), telegram-send, email-digest, impact-graph (and networkx), ingest-run, cleanup-old-signals; WebSocket for real-time signal updates
  • Export and notifications: Telegram digest, send to Telegram, Excel/PDF, email digest

Local / dev: Go dashboard + Python API

Locally you can run the Python API (ingest, graph, Telegram, NVIDIA generation) and a lightweight Go dashboard with the same design and fast response. One SQLite DB: Python writes signals and graph; Go reads and serves the dashboard and API.
Component Technology
Python API FastAPI, port 8080; analysis generation via NVIDIA API (NVIDIA_API_KEY in .env)
Go dashboard Go 1.21+, Gin, port 8090
DB SQLite (e.g. signals.db), shared by Python and Go
AI in Go Go code may use DeepSeek (DEEPSEEK_API_KEY). In production only NVIDIA via Python; for parity call Python API or add NVIDIA to Go.

Go project structure

go-signal-analysis/: main.go, handlers/ (dashboard, signals, graph, analysis, analysis_page), models/signal.go, database/db.go, templates/dashboard.html, static/

Go API (port 8090)

Method Endpoint Description
GET / Redirect to /dashboard
GET /dashboard Dashboard (HTML)
GET /api/signals Signals list (sector, region, min_impact, min_confidence, date_from, limit)
GET /api/stats Aggregated stats
GET /api/graph-data Graph data (optional ?signal_id=...)
POST /api/generate-analysis/:signal_id Generate analysis (body: {"language":"ru"} or "en") — in Go via DeepSeek
GET /generate-analysis/:signal_id Analysis page

Quick start (local)

1. Python API (from repo root with .env including NVIDIA_API_KEY): python -m uvicorn app:app --host 0.0.0.0 --port 8080 2. Go dashboard: cd go-signal-analysis, go mod tidy, go run main.go — dashboard at http://localhost:8090/dashboard, API at /api/signals, /api/stats, /api/graph-data. Build binary: go build -o signal-analysis-server main.go.
