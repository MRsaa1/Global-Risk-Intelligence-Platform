# Yoast SEO Advanced Settings - Инструкция

## Настройка дополнительных параметров SEO для SAA Alliance Landing Page

---

## Рекомендуемые настройки

### 1. Позволить поисковым системам показывать это содержимое в результатах поиска?

**✅ Рекомендуемое значение:** **Yes (Да)**

**Почему:**
- Landing page должна индексироваться поисковыми системами
- Это необходимо для появления в результатах поиска Google, Yandex и других
- Если выбрать "No", страница не будет показываться в поиске

**Действие:**
- Выберите **"Да (значение по умолчанию для типа)"** или **"Yes"**

---

### 2. Должны ли поисковые системы проходить по ссылкам этого содержимого?

**✅ Рекомендуемое значение:** **Yes**

**Почему:**
- Landing page содержит важные ссылки на платформы SAA
- Поисковые системы должны индексировать эти ссылки
- Это помогает передавать "link juice" на связанные страницы

**Действие:**
- Выберите радиокнопку **"Yes"**

---

### 3. Расширенная настройка тега Meta Robots

**✅ Рекомендуемое значение:** **Оставить пустым** (или выбрать "index, follow")

**Почему:**
- Для стандартной landing page не нужны специальные настройки
- Если оставить пустым, будут использоваться настройки из пунктов 1 и 2
- Можно выбрать "index, follow" для явного указания

**Действие:**
- Оставьте поле **пустым**
- Или выберите "index, follow" из выпадающего списка (если доступно)

---

### 4. Заголовок для цепочек навигации (Breadcrumbs Title)

**✅ Рекомендуемое значение:** `Scientific Analytics Alliance` или оставить пустым

**Почему:**
- Breadcrumbs помогают пользователям и поисковикам понимать структуру сайта
- Если breadcrumbs отключены (как в настройках Astra), можно оставить пустым
- Если включены - лучше указать короткий заголовок

**Действие:**
- Введите: `Scientific Analytics Alliance`
- Или оставьте **пустым**, если breadcrumbs не используются на сайте

---

### 5. Канонический URL-адрес (Canonical URL)

**✅ Рекомендуемое значение:** Полный URL страницы

**Почему:**
- Canonical URL указывает поисковым системам основную версию страницы
- Предотвращает проблемы с дублированием контента
- Должен быть полным URL с протоколом https://

**Действие:**
- Введите полный URL страницы
- Примеры:
  - `https://saa-alliance.com/scientific-analytics-alliance/`
  - `https://saa-alliance.com/` (если это главная страница)
- **Как узнать правильный URL:**
  1. Откройте страницу на сайте
  2. Скопируйте URL из адресной строки браузера
  3. Вставьте в поле Canonical URL

## Пошаговая инструкция

### Шаг 1: Откройте раздел "Дополнительно"

1. В WordPress редакторе найдите блок **Yoast SEO** (обычно внизу страницы)
2. Найдите вкладку или секцию **"Дополнительно"** (Advanced)
3. Нажмите на заголовок чтобы развернуть секцию (если свернута)

### Шаг 2: Настройте индексацию

1. Найдите вопрос: **"Позволить поисковым системам показывать это содержимое в результатах поиска?"**
2. В выпадающем списке выберите:
   - **"Да (значение по умолчанию для типа)"** 
   - Или **"Yes"** (если доступно)

### Шаг 3: Настройте следование по ссылкам

1. Найдите вопрос: **"Должны ли поисковые системы проходить по ссылкам этого содержимого?"**
2. Выберите радиокнопку **"Yes"**
3. Убедитесь что "No" НЕ выбрано

### Шаг 4: Расширенные настройки Meta Robots

1. Найдите поле: **"Расширенная настройка тега Meta Robots"**
2. Оставьте поле **пустым**
3. Или выберите "index, follow" из выпадающего списка (если доступно)

### Шаг 5: Заголовок для breadcrumbs

1. Найдите поле: **"Заголовок для цепочек навигации"**
2. Введите: `Scientific Analytics Alliance`
3. Или оставьте пустым, если breadcrumbs отключены

### Шаг 6: Canonical URL

1. Найдите поле: **"Канонический URL-адрес"**
2. Откройте страницу на сайте в новой вкладке
3. Скопируйте полный URL из адресной строки (например: `https://saa-alliance.com/scientific-analytics-alliance/`)
4. Вставьте URL в поле Canonical URL
5. Убедитесь что URL начинается с `https://` и заканчивается на `/`

### Шаг 7: Сохраните изменения

1. Нажмите кнопку **"Обновить"** в правом верхнем углу страницы
2. Дождитесь сообщения об успешном обновлении
3. Проверьте что все настройки сохранены

## Проверка после настройки

### 1. Проверка в исходном коде страницы

1. Откройте страницу на сайте (не в админке)
2. Нажмите **Ctrl+U** (Windows) или **Cmd+U** (Mac) для просмотра исходного кода
3. Найдите в секции `<head>` следующие теги:

**Должно быть:**
```html
<meta name="robots" content="index, follow">
<link rel="canonical" href="https://saa-alliance.com/scientific-analytics-alliance/">
```

**Если видите:**
- `content="noindex"` - значит индексация отключена (НЕПРАВИЛЬНО)
- `content="nofollow"` - значит ссылки не индексируются (НЕПРАВИЛЬНО)
- Нет canonical тега - нужно добавить URL

### 2. Проверка в Google Search Console

1. Войдите в Google Search Console
2. Используйте инструмент "Проверка URL"
3. Введите URL вашей страницы
4. Проверьте что:
   - Страница доступна для индексации
   - Canonical URL правильный
   - Нет ошибок

## Частые вопросы

### Вопрос: Что если я выберу "No" для индексации?

**Ответ:** Страница не будет показываться в результатах поиска. Это плохо для landing page, так как никто не сможет найти страницу через поиск.

### Вопрос: Нужно ли заполнять Canonical URL если страница одна?

**Ответ:** Да, рекомендуется всегда указывать canonical URL. Это помогает поисковым системам понимать, какая версия страницы основная, даже если есть дубликаты.

### Вопрос: Что делать если breadcrumbs отключены?

**Ответ:** Поле "Заголовок для цепочек навигации" можно оставить пустым. Оно используется только если breadcrumbs включены в теме.

### Вопрос: Можно ли оставить Meta Robots пустым?

**Ответ:** Да, можно. Если оставить пустым, будут использоваться настройки из пунктов 1 и 2 (индексация и следование по ссылкам).

## Дополнительные рекомендации

### Если страница не индексируется:

1. Проверьте настройки в Google Search Console
2. Убедитесь что robots.txt не блокирует страницу
3. Проверьте что в Yoast стоит "Yes" для индексации
4. Убедитесь что страница не в режиме "Черновик" или "Частная"

### Canonical URL для разных версий сайта:

- Если есть версии с `www` и без `www` - выберите одну предпочтительную
- Если есть версии с `/` и без `/` в конце - выберите одну предпочтительную
- Обычно выбирают версию **без www** и **с завершающим слешем**

Примеры правильных canonical URL:
- ✅ `https://saa-alliance.com/scientific-analytics-alliance/`
- ❌ `http://saa-alliance.com/scientific-analytics-alliance` (без https и слеша)
- ❌ `www.saa-alliance.com/scientific-analytics-alliance/` (с www, если предпочтительна версия без www)
